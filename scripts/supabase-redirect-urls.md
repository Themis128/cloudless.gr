# How to Set Supabase Redirect URLs for Local and LAN Access

To allow authentication and OAuth login from both `http://localhost:3000` and your LAN IP (e.g., `http://192.168.0.23:3000`), update your Supabase project settings as follows:

## 1. Update Supabase Auth Redirect URLs

1. Go to your [Supabase Project Dashboard](https://app.supabase.com/).
2. Select your project.
3. Navigate to **Authentication** > **URL Configuration** (or **Settings** > **Auth Settings** > **Redirect URLs**).
4. In the **Site URL** and **Redirect URLs** fields, add the following (one per line):

   ```
   http://localhost:3000
   http://localhost:3000/*
   http://192.168.0.23:3000
   http://192.168.0.23:3000/*
   ```

5. Click **Save**.

## 2. Update OAuth Provider Redirect URLs (Google, GitHub, etc.)

For each OAuth provider you use:

1. Go to the provider's developer console (e.g., Google Cloud Console, GitHub Developer Settings).
2. Find your OAuth app for this project.
3. Add both of these as **Authorized Redirect URIs**:

   ```
   http://localhost:3000/auth/callback
   http://192.168.0.23:3000/auth/callback
   ```

4. Save changes.

## 3. Test Login from Both Addresses

- Open your app at `http://localhost:3000` and `http://192.168.0.23:3000` from your browser or another device on your LAN.
- Login and OAuth flows should work from both addresses.

---

**Tip:** If your LAN IP changes, update the URLs in Supabase and your OAuth provider settings accordingly.
