# How to Set Supabase Redirect URLs Locally via Docker

If you are running Supabase locally (e.g., with Docker Compose), you can set redirect URLs for authentication and OAuth by editing your `.env` file:

## 1. Open Your Local Supabase Project Directory

This is the folder where your `docker-compose.yml` file lives.

## 2. Edit the `.env` File

- If `.env` does not exist, create it in the same directory as `docker-compose.yml`.
- Add or update the following lines:

```env
SITE_URL=http://localhost:3000
AUTH_REDIRECT_URLS=http://localhost:3000,http://localhost:3000/*,http://192.168.0.23:3000,http://192.168.0.23:3000/*
```

- You can add more URLs separated by commas if needed.

## 3. Restart Supabase Docker Services

After saving the `.env` file, restart your Supabase stack for changes to take effect:

```sh
docker-compose down
docker-compose up -d
```

## 4. Test Login from Both Addresses

- Open your app at `http://localhost:3000` and `http://192.168.0.23:3000`.
- Login and OAuth flows should work from both addresses.

---

**Tip:** If your LAN IP changes, update the `.env` file and restart Docker.
